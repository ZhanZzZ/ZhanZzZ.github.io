2023 年度总结

整体概况

本人情况

  • 大专学历
  • 开发实际工龄两年半不到
  • 今年 27

工作

23 年整体工作方面怎么说呢，整体看来是比较正向的。

从一家加班十分恐怖，通勤来回 40 分钟左右的公司跳到了一家半年只加了 3 天半，通勤来回 15 分钟的公司，涨薪 23% 左右，算是比较幸运了。

离职

前年入职上家公司的时候怎么说呢，其实是个人处于一个不是很好的状态里的。

  1. 本身实际入行也就一年的时间，包装工龄之后对面试没什么底
  2. 找工作有点慌急慌忙的，其实也没有很好的做准备

不过有在上上家公司好好工作，仗着自己面试的时候气场比较稳定倒是也入职了。

入职之后慢慢的发觉这里的加班情况好像和入职前说的不太一样。

面试的时候说的只要在项目上线前加加班，入职后发现公司是外包项目，基本好几个项目穿插着处理。再加上人少嘛，有时候别的项目忙不过来要去帮忙，那上线的时候你自然是要在的，所以面对的是非常频繁的加班。

本来一入职一个多月发现不对劲之后就想离职了，但是由于我之前工作不太稳定，对象劝我稳一稳再说，所以也就咬咬牙顶住了。后来干了半年，身子实在不舒服去检查了一下，发现之前身体的毛病变严重了，就直接走了钉钉流程润了。

入职

其实这次找工作算是运气很好了。当时因为身体原因住院了，在医院没事做刷刷 boss 看看大体行情如何。离职前一直收到的讯息是「今年不好找工作」、「大环境不好」诸如此类的。所以想着带着看看，反正闲着也是闲着。

至于面试题呢，倒是加了几个卖课的，白嫖了面试题。计划里是在医院的时候好好看看题目，背一背，出院了之后找工作刚好直接投简历就行了。不过看了半小时之后发现根本看不下去，后面也懒得看了，直接开摆。

正如一开始说的，我运气很好。大概两个礼拜左右，我就找到工作并入职了，公司离家近且涨薪 23%。

生活

置换

这一年最大的感受就是心态变得更好了，并且对「置换」有了更深的理解。

比如你在公司好好工作，那就是用你的时间和精力去置换金钱；在家陪伴自己的家人，那就是用自己的时间和情感去置换家人之间的羁绊；学习就是用时间去置换一些技能。不管事情都是置换，无非是赚了或者亏了。

学习

年底了，学习一点劲都没有。23 年也只是把 java 入了个门，会写一些 CRUD 了。

经常看到群里一些群友每天都在学习新东西，什么 Go、RN、Python 等等，其实挺费解的。如果能用上，或者说以后方向设计这门语言，那去学一下倒也无可厚非。但是仅仅为了技多不压身去学一门新的语言感觉挺浪费时间的。深入的学不到，仅仅是入门挺没意思的，而且就像是 React 之于 Vue ，会其中一个之后去学另一个也是很快的，因为核心思想其实是一致的。

就我个人而言，学个 java 也就够了。如果后面需要使用别的，那到时候再学就好了，影响也不大。

前后端

目前感觉前后端对我来说区别并不大了，就像上面说的，我有写后端的需求，所以去学个后端。现在自己有一些小点子，小想法直接前后端自己梭哈了。那现在我到底算是前端还是后端呢？

可能有的朋友会说这不就是全干（全栈）工程师嘛？

但这不就是一开始的软件开发嘛，那有什么前后端的说法。

所以说一千，道一万，就是以需求为导向去开发软件。

学习的目的

看书、看学习视频、背单词…诸如此类的行为根本目的是为了学习新的知识，让自己变得更好。但是有些朋友很奇怪，变成了一种「打卡」行为。

要以此为鉴，明白自己看书，背单词的目的到底是为了什么。如果总是以打卡心态去学习，那只能是感动自己

明年目标

多输出文章，提高表达能力

自己的表达能力其实挺有限的，比如上面的内容，想说的很多，又不知道怎么说。看上去很怪，又不知道怎么优化。希望明年可以好好提高一下自己的表述能力，不管是口头上还是文字上的，都能进步

英语

今年背单词的计划算是寄了，年终沉迷 lolm，年末沉迷老头环、小骨，只有年初的时候背了一段时间。希望 24 年可以好好背单词，学英语。

技术

今年只看了 lodash 差不多十分之一出头的源码，真难顶。24 年的话前端把以前刻意逃避的 Proxy 这些好好过一下，看看源码，顺道把 java 也提升一下
